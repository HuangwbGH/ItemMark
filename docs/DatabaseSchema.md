## 1. 数据库组成

ItemMark 当前使用 SQLite 保存模块缓存数据和管理数据，并通过 U8 SQL Server 查询源数据或库存台账。

## 2. 模块业务库

模块业务库路径：

```text
data/material-info/itemmark.db
data/wuping/itemmark.db
```

业务表沿用旧项目结构：

```text
material_label_items
```

用途：

- 缓存物料基础信息。
- 保存模块二维码 URL。
- 支持标签打印页、扫码详情页和物料搜索 API。

## 3. 管理库

管理库路径：

```text
data/admin/itemmark_admin.db
```

主要职责：

- 保存模块配置。
- 保存同步配置。
- 保存同步运行记录。

设计表：

```text
module_configs
sync_configs
sync_runs
```

字段设计以 `docs/02-程序设计实现方案.md` 的数据库设计章节为准。

## 4. 台账查询数据源

武平模块库存台账不只依赖 SQLite 缓存，扫码页和 API 会按模块配置实时查询 U8 账套：

```text
UFDATA_105_2019
```

台账 SQL 参考：

```text
docs/20260513库存台账查询变更.md
```

## 5. 维护规则

- 数据库结构变更必须同步更新本文档和技术设计文档。
- 生产数据、真实密码和隐私数据不得提交到文档。
- 新增迁移脚本时应放在 `scripts/` 或明确记录执行方式。
