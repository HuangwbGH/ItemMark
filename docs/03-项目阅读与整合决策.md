# 两个旧项目阅读与整合决策

状态：已根据补充信息调整，待确认后实施

## 1. 项目 1：material-label-system

定位：对外物料信息模块的基础。

已确认价值：

- 项目 1 已经在 Linux OS 上正常运行。
- 项目 1 的二维码和标签模板已经调试好，可以正常使用。
- 项目 1 扫码后查询物料基础信息。
- 项目 1 的旧二维码路径需要优先兼容。

整合决策：

- 项目 1 不再作为独立项目名保留，整合为 ItemMark 的“对外物料信息模块”。
- 保留 `/material/{code}` 兼容旧二维码。
- 复制项目 1 的标签配置作为 `config/templates/material-info/label_config.xlsx`。
- 可复制项目 1 的真实 `.env`、数据库、证书私钥作为部署资料。

## 2. 项目 2：wupingPMC

定位：武平模块的基础。

已确认价值：

- 项目 2 已实现扫码查询库存台账。
- 项目 2 支持标签字段“单位”。
- 项目 2 已有 U8 台账 SQL 和手机详情页。
- 项目 2 的二维码必须和项目 1 区分。

整合决策：

- 项目 2 不再作为独立项目名保留，整合为 ItemMark 的“武平模块”。
- 武平模块二维码使用 `/wuping/material/{code}`。
- 武平模块打印页、批量打印页、二维码接口都使用 `/wuping` 前缀。
- 复制项目 2 的台账查询代码和模板。

## 3. 总体整合决策

最终方案：

- 一个项目：ItemMark。
- 一个域名：`https://code.zestrade.com`。
- 多个模块：`material-info`、`wuping`。
- 模块独立配置：数据库地址、账套、账号、密码、同步计划、模板、二维码路径。
- 模块独立二维码：不同模块打印出的二维码扫码结果不同。
- 对外物料信息模块路径确认使用 `/material-info`。
- 旧项目 1 当前生产二维码确认是 `https://code.zestrade.com/material/{code}`，无额外端口或路径前缀。
- 后台同步和模块配置页面首版不需要登录保护。
- 后台路径首版暂不做 Nginx IP 白名单。
- 物料基础信息虽然通用，首版仍保留模块 SQLite 隔离。
- 程序运行路径全部使用相对路径。
- ItemMark 应用服务端口使用非常用端口，首版为 `18089`。
- 实现阶段遵循 `2026-04-20-LLM-写代码规范.md`。
- 首次运行重新同步。
- 提供后台同步管理页面。

## 4. 路径决策

保留兼容：

```text
https://code.zestrade.com/material/{code}
```

用途：对外物料信息模块扫码页，兼容旧项目 1。

新增模块路径：

```text
https://code.zestrade.com/material-info/...
https://code.zestrade.com/wuping/...
```

武平二维码：

```text
https://code.zestrade.com/wuping/material/{code}
```

## 5. 默认配置决策

默认 U8 配置：

| 配置 | 默认值 |
|---|---|
| 地址 | `192.168.1.135` |
| 端口 | `1433` |
| 对外物料信息模块账套 | `UFDATA_104_2018` |
| 武平模块账套 | `UFDATA_105_2019` |
| 账号 | `testreadonly` |

说明：

- 这些只是默认值。
- 每个模块都可以分别修改数据库地址、账套、账号和密码。
- 物料基础信息在所有账套通用；104 是集团主账套、贸易账套，105 是集团下子公司工厂账套。
- 扫码效果不同的关键不是物料基础信息不同，而是不同账套的出入库和库存结存不同。

## 6. 同步决策

同步能力从“命令行定时任务”升级为“后台可管理任务”。

需要实现：

- 每个模块独立同步。
- 每个模块独立同步时间和频率。
- 页面手动触发同步。
- 页面查看同步状态和历史结果。
- 首次运行重新同步。

## 7. 模板决策

每个模块独立模板：

```text
config/templates/material-info/label_config.xlsx
config/templates/wuping/label_config.xlsx
```

决策：

- 对外物料信息模块复制项目 1 已调试好的模板。
- 武平模块复制项目 2 模板，并确保二维码路径指向 `/wuping/material/{code}`。
- 后续新增模块时新增自己的模板目录。

## 8. 安全和迁移决策

根据补充信息，允许复制旧项目真实资料：

- `.env`
- 数据库文件
- 证书私钥

实施注意：

- 可以复制到 ItemMark 目录用于部署。
- 部署前仍建议备份旧项目目录和旧 Nginx 配置。
- 日志和页面不展示明文密码。
- 证书文件权限需要在 Linux OS 上保持最小可读。

## 9. 已确认事项

1. 旧项目 1 当前生产二维码是 `https://code.zestrade.com/material/{code}`。
2. 对外物料信息模块后台路径使用 `/material-info`。
3. 武平模块默认账套使用 `UFDATA_105_2019`。
4. 同步管理和模块配置页面首版不需要登录保护。
5. 物料基础信息在所有账套通用；不同模块扫码效果不同，是因为不同账套的出入库和库存结存不同。

## 10. 最终实施确认

1. 首版保留两个模块各自 SQLite，不做共享物料主数据表。
2. 首版后台不做应用登录，也暂不做 Nginx `/admin` IP 白名单。
3. 所有运行配置路径使用相对路径。
4. 应用端口不使用常用端口，首版使用 `18089`。
5. 实现前必须遵循指定开发规范，先明确假设、成功标准和验证方式。
6. 如果后续后台暴露风险变高，再补登录、IP 白名单或 VPN 访问限制。
